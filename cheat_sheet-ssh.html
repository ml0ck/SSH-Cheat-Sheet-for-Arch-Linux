SSH Cheat Sheet for Arch Linux

Everything you need to know about SSH and its usage on Arch Linux

What is SSH?

SSH (Secure Shell) is a protocol that allows secure remote login to another computer over a network. It uses encryption to protect the data transferred between the client and the server, ensuring the confidentiality and integrity of the communication.

Install SSH Server on Arch Linux

To install the OpenSSH server package on Arch Linux, follow these steps:

sudo pacman -S openssh  # Install OpenSSH server
sudo systemctl enable sshd  # Enable SSH service at boot
sudo systemctl start sshd   # Start SSH service

Start and Enable SSH Service

To start and enable the SSH service on Arch Linux:

sudo systemctl start sshd  # Start SSH service
sudo systemctl enable sshd  # Enable SSH service on boot

SSH Command Syntax

Use SSH to connect to a remote server with the following syntax:

ssh [user]@[hostname_or_ip_address]  # Basic SSH command

SSH Key-Based Authentication

To use key-based authentication for a more secure login process:

# Generate SSH keys
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
# Copy the public key to the server
ssh-copy-id user@hostname_or_ip
# SSH login with keys
ssh user@hostname_or_ip

Common SSH Options

Here are some useful SSH options:

# Run a command on the remote server
ssh user@hostname_or_ip "command_to_run"

# Use a specific port
ssh -p 2222 user@hostname_or_ip

# Disable strict host key checking
ssh -o StrictHostKeyChecking=no user@hostname_or_ip

Troubleshooting

Common SSH issues and troubleshooting steps:

# Check SSH status
sudo systemctl status sshd

# Restart SSH service
sudo systemctl restart sshd

# Check SSH connection issues
ssh -v user@hostname_or_ip  # Verbose mode for debugging

30 Useful SSH Commands and Tips

  • ssh -p [port] user@hostname # Connect on a specific port
  • ssh -i [private_key] user@hostname # Specify a private key
  • scp file.txt user@hostname:/path/to/destination # Copy file via SSH
  • sftp user@hostname # Interactive file transfer over SSH
  • ssh user@hostname 'command' # Execute a remote command
  • ssh user@hostname 'ls -l' # List remote directory
  • ssh-copy-id user@hostname # Copy SSH public key to server
  • ssh -X user@hostname # Enable X11 forwarding for GUI apps
  • ssh -A user@hostname # Enable agent forwarding
  • ssh -v user@hostname # Debug SSH connection
  • ssh user@hostname 'mkdir -p /path' # Create directories on remote machine
  • sshfs user@hostname:/remote/path /local/path # Mount remote directories
  • ssh -L [local_port]:[remote_host]:[remote_port] user@hostname # Port forwarding
  • ssh -R [remote_port]:[local_host]:[local_port] user@hostname # Reverse port forwarding
  • ssh -C user@hostname # Enable compression for faster transfers
  • ssh -q user@hostname # Quiet mode (no output)
  • ssh user@hostname 'tail -f /var/log/syslog' # View logs remotely
  • ssh user@hostname 'ps aux' # View processes on remote system
  • ssh user@hostname 'free -m' # View memory usage on remote system
  • ssh user@hostname 'df -h' # Check disk usage remotely
  • ssh user@hostname 'uptime' # Check uptime on remote server
  • ssh user@hostname 'who' # See who is logged in on the remote server
  • ssh user@hostname 'cat /etc/os-release' # View OS version
  • ssh -t user@hostname 'sudo reboot' # Reboot remote server
  • ssh user@hostname 'shutdown -h now' # Shutdown remote server
  • ssh -o ConnectTimeout=10 user@hostname # Set connection timeout
  • ssh -o UserKnownHostsFile=/dev/null user@hostname # Disable known hosts check
  • ssh -o PreferredAuthentications=password user@hostname # Force password authentication
  • ssh user@hostname 'uname -a' # Get system information
  • ssh -t user@hostname 'exit' # Exit remote session after command
  • ssh user@hostname 'top' # View system resource usage remotely
  • ssh user@hostname 'dmesg' # View kernel messages
